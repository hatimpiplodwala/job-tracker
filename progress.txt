Known limitations / v1 trade-offs
---------------------------------
- No email verification or password reset (by design, per PRD)
- Status validation in Python only (no DB CHECK constraint)
- Edit button per row (Streamlit has no native row-click event)
- supabase pinned to <2.16 due to pyiceberg build failure on Python 3.14

Possible future improvements (v2)
----------------------------------
- Swap Streamlit for React/Next.js for stronger frontend signal on resume
- Add AI feature: auto-parse job description URL → fill form fields
- Email/reminder system for follow-ups
- Kanban board view
- Browser extension to add jobs in one click
- FastAPI layer between frontend and Supabase
